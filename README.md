# Code for "Why quantum machine learning can be harder to evade under adaptive attacks"

**Authors:** Xie, Feng    
**Data deposit:** https://doi.org/10.5281/zenodo.20700154

---

## Overview

This repository contains all code needed to reproduce the experiments and figures
in the paper. The core finding is that a structural property of Pauli measurement
operators — captured in Proposition 1 — prevents adaptive adversaries from
simultaneously achieving backdoor success and Mahalanobis detector evasion in the
tested frozen-head amplitude-encoded QML architecture.

**No model weights are distributed.** Baseline model weights are generated by
`training/scb_mnist_baseline.py` / `training/scb_bloodmnist_baseline.py`.
Adaptive-attack weights are generated by `training/train_adaptive_qnn_fast.py`.
Pre-computed metric CSVs are available at the Zenodo deposit above.

## Reproduction pipeline

### 0. Install dependencies

```bash
pip install -r requirements.txt
```

### Option A — Verify from pre-computed data (fastest, no GPU needed)

Download the Zenodo deposit (DOI above) and unpack it so that the `data/`
folder sits at the repository root, then:

```bash
python reproduce_key_numbers.py
# writes reproduction_outputs/key_numbers_recomputed.csv
# all 34 assertions should pass
```

---

### Option B — Full end-to-end reproduction (requires Linux server + CPU/GPU)

#### Step 1a — Baseline training: MNIST

Single configuration (quick test — reproduces paper's seed 42, L=12, t7_vs_t0 result):
```bash
python training/scb_mnist_baseline.py \
    --random-seed 42 --n-seeds 1 \
    --target-class 7 --non-target-class 0 \
    --n-qubits 5 --n-layers 12 \
    --epsilon 0.8 --poison-ratios 0.1 \
    --apply-pca --standardize \
    --output-dir results/baseline_mnist/seed_42/layer_12_grid/t7_vs_t0/pr_0.1
```

Expected output (results_summary_table_fixed_upasr.csv):
```
CA ≈ 97.5%,  ASR ≈ 70.0%,  AUC ≈ 0.943
```

> **Note:** BloodMNIST dataset (`bloodmnist.npz`) is downloaded automatically to `data/`
> on first run. If auto-download fails due to network issues, download manually from
> https://zenodo.org/records/10519652 and place in `data/`.

Full grid (seeds 42-46 × layers 3-12 × 9 non-target classes × 5 PR values = 2250 tasks):
```bash
bash training/run_baseline_mnist.sh --max-jobs 16
# estimated ~24 h on 16 CPU cores
```

#### Step 1b — Baseline training: BloodMNIST

Single configuration (quick test — reproduces paper's seed 42, L=8, t6_vs_t0 result):
```bash
python training/scb_bloodmnist_baseline.py \
    --random-seed 42 --n-seeds 1 \
    --target-class 6 --non-target-class 0 \
    --n-qubits 8 --n-layers 8 \
    --epsilon 0.8 --poison-ratios 0.1 \
    --encoding amplitude --n-epochs 200 \
    --output-dir results/baseline_bloodmnist/seed_42/layer_8_grid/t6_vs_t0/pr_0.1
```

Expected output (models/clean_model_seed_42_epoch_200.pt written; validation CA ≈ 76–78%).
Attack metrics (ASR, AUC) are produced in Step 2.

Full grid (seeds 42-46 × layers 3-12 × 6 non-target classes × 5 PR values = 1500 tasks):
```bash
bash training/run_baseline_bloodmnist.sh --max-jobs 8
# estimated ~48 h on 8 CPU cores (8-qubit circuits are slower)
```

#### Step 2 — Adaptive attack training

Single run:
```bash
python training/train_adaptive_qnn_fast.py \
    --seed 42 --layer 12 --lambda_r 1.0 --dataset mnist \
    --data-root results/baseline_mnist \
    --out-root  results/training_time_mnist
```

Full sweep (5 seeds × 4 λ_r × 2 datasets = 40 tasks):
```bash
bash training/run_sweep.sh    # edit MAX_JOBS to match your server
```

#### Step 3 — Analysis (generate processed CSVs)

Run all analysis scripts from the repository root.  
Each script reads from the `results/` directories produced in Steps 1-2
and writes to `data/` (the same layout as the Zenodo deposit):

```bash
python analysis/p05_indep_calibration_split.py --dataset mnist
python analysis/p05_indep_calibration_split.py --dataset blood
python analysis/p05_jser_dc.py
python analysis/p05_repeated_splits_full.py
python analysis/pauli_ladder_ablation.py --dataset mnist
python analysis/pauli_ladder_ablation.py --dataset blood
python analysis/e2_nullspace_attack.py
python analysis/a7_shot_sweep_adaptive.py
python analysis/classical_bloch_control.py
python analysis/classical_bloch_small_ablation.py
python analysis/a1b_bloodmnist_unified_metrics.py
```

#### Step 4 — IBM Quantum hardware results

Download `job-d8b9auh59p8c73bm6mtg.zip` and `job-d8be0btmdd1s73b9e2bg.zip`
from the Zenodo deposit into `data/hardware/`, then:

```bash
python hardware/parse_ibm_hw_results.py
```

#### Step 5 — Verify key numbers

```bash

```

#### Step 6 — Generate figures

```bash
python figures/figures_all_v2.py
```

## Shared dependency: fast_circuit.py

`fast_circuit.py` is a pure-numpy statevector simulator that supports
AmplitudeEmbedding (MNIST, 5 qubits) and AngleEmbedding (BloodMNIST, 8 qubits).
It exposes:

- `run_circuit_batch(samples, weights, n_qubits, n_layers)` — batch XYZ features
- `_make_evals(weights, n_qubits, n_layers)` — return a callable `φ(x)`

All analysis scripts in `analysis/` import from `fast_circuit.py` in the repository
root via `sys.path` insertion; no installation step is required.

`qiskit-ibm-runtime` (see `requirements.txt`), tested separately.
